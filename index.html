Toolkit

Six tools your AI didn't know it needed

S

c3_search

TF-IDF + optional semantic embeddings via Ollama. Finds files by meaning, not just text matching. Sub-second results across any codebase.

M

c3_compress

Structural maps that expose every class, function, and signature for the cost of a few tokens. 340 lines becomes a 28-token overview.

R

c3_read

Symbol-level extraction. Pull one function from a 2,000-line file. Read 22 lines instead of 2,000. Surgical precision.

V

c3_validate

Native syntax checkers for 15+ languages including Python, JS, TS, Java, Go, Rust, R, PHP, Ruby. Background validation with result caching.

P

c3_memory

Persistent fact store with auto-learning from tool calls. Your AI remembers decisions, patterns, and conventions across sessions automatically.

F

c3_filter

Pre-processes logs, terminal output, and data files. Extracts signal from noise before it ever hits the AI's context window.

Benchmark Methodology

E2E Agent Benchmark: 14 tasks across 7 categories (easy to expert). Each task runs Claude Sonnet 4.6 twice on an identical git worktree sandbox: once with C3 MCP tools, once without. Responses are scored across five deterministic dimensions (keyword, structural, file mention, factual, completeness) using ground-truth evaluation. No cherry-picking, no prompt tuning between runs. C3 won 8 of 14 tasks (57%), with a weighted win rate of 69% accounting for score magnitude.

Token Savings Benchmark: 88 source files measured across 8 tool categories (compress, map, read, validate, search, filter, delegate). Raw tool-level multiplier: 9.87x. After accounting for C3's own overhead (mandates, schemas, mandatory recalls), session-net savings are 75.9% (balanced) to 83.0% (heavy analysis). Full source code and raw JSON results are included in the repository.
